## Файлы seatop в директории input

В директории `sway/input/` обычно есть несколько файлов, связанных с seatop, например:

- `seatop.h` — заголовочный файл с описанием интерфейса seatop.
- `seatop_default.c` — реализация "дефолтного" seatop, который обрабатывает обычное управление курсором и окнами.
- `seatop_move.c` — реализация seatop для перемещения окна.
- `seatop_resize.c` — реализация seatop для изменения размера окна.
- Возможно, другие seatop для специальных режимов (например, для drag-and-drop, для ввода текста и т.п.).

---

## Как они работают вместе?

1. **Начало seatop**: Когда пользователь начинает действие, например, нажимает кнопку мыши на заголовке окна, вызывается `seatop_begin()`, который переключает текущий seatop на нужный (например, `seatop_move`).
    
2. **Обработка событий**: Все события ввода (движение мыши, нажатия кнопок, прокрутка) перенаправляются в функции текущего seatop, например, `seatop_move_pointer_motion()`, `seatop_move_button()`. Эти функции реализуют логику перемещения окна.
    
3. **Завершение seatop**: Когда действие заканчивается (например, отпускается кнопка мыши), вызывается `seatop_end()`, и seat возвращается в дефолтный режим (`seatop_default`).
    

---

## Основные методы seatop и их назначение

В `seatop.h` определён интерфейс с такими методами (пример):

- `button(seat, time, button, state)` — обработка нажатия или отпускания кнопки мыши.
- `pointer_motion(seat, time, dx, dy)` — обработка относительного движения мыши.
- `pointer_motion_absolute(seat, time, x, y)` — обработка абсолютного движения (например, с тачскрина).
- `axis(seat, time, orientation, delta, discrete)` — обработка прокрутки колёсика мыши.
- `frame(seat)` — вызов после серии событий ввода (например, для обновления состояния).
- `switch_toggle(seat, time, switch_code, state)` — обработка переключателей (например, переключение клавиатурных режимов).
- `cancel(seat)` — отмена текущей операции seatop (например, при нажатии Esc).
- `commit(seat)` — подтверждение и завершение операции.

## Что делают конкретные seatop?

### seatop_default

- Обрабатывает обычное поведение курсора.
- Перенаправляет клики и движения мыши на окна.
- Запускает другие seatop при необходимости (например, при начале перетаскивания).

### seatop_move

- Обрабатывает перемещение окна.
- При движении мыши изменяет позицию окна.
- Завершается при отпускании кнопки мыши.

### seatop_resize

- Обрабатывает изменение размера окна.
- При движении мыши изменяет размеры окна в зависимости от выбранного края или угла.
- Завершается при отпускании кнопки мыши.

---

## Где применяются?

- Вся логика взаимодействия пользователя с окнами и курсором в Sway проходит через seatop.
- Это ключевой механизм для реализации drag-and-drop, ресайза, перемещения, а также других интерактивных действий.
- Позволяет легко расширять и модифицировать поведение ввода, добавляя новые seatop.

## Итог

- **seatop** — это модуль, который управляет состоянием и логикой обработки ввода для seat.
- В Sway есть несколько реализаций seatop для разных режимов работы с окнами.
- Методы seatop обрабатывают события ввода и управляют поведением окон и курсора.
- Переключение между seatop происходит динамически в зависимости от действий пользователя.
